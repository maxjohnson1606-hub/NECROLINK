1. Persistent Data (Most Important)

Nothing on the website should reset after refreshing the page.

Every change made by the Owner or Admin must be permanently saved in the database.

This includes:

News
Events
Clan members
Teams
Player profiles
User accounts
Roles
Permissions
Store items
Shop inventory
Gallery
Tournament data
Match history
Recruitment status
Announcements
Home page edits
Hero tier lists
Rankings
Achievements
Social links
Contact information
Settings
Banners
Logos
Images
Theme settings
Navigation menu
Everything editable on the website

Refreshing the page must NEVER restore default values.

The website must always load the latest saved data from the database.

Use a real backend database (Supabase, Firebase Firestore, PostgreSQL, or another production database). Do NOT use localStorage, sessionStorage, mock data, or temporary state for production data.
